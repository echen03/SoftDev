LESS

A CSS Preprocessor

By Team Eggcellent: Tiffany Cao and Ethan Chen

What is LESS?

LESS, which stands for Leaner Style Sheets, is a style sheet language that implements the use of coding to extend the capabilities of CSS. Essentially, it is like a cross between Python and CSS, so that it is easier to maintain and customize your style sheets.

How to Use

A CSS preprocessor has two parts:
  1. the language
  2. the compiler

The language is the LESS code itself, while the compiler is what turns LESS code into standard CSS. Compilers include Ruby Gem, CodeKit (an OS X app), less.js, and more.

Using less.js is convenient because it essentially works the same way as a CSS stylesheet.

<link rel = "stylesheet/less" type = "text/css" href = "styles.less"/>
<script src = "//cdnjs.cloudflare.com/ajax/libs/less.js/3.9.0/less.min.js"></script>

Another script you can use is:

Features of LESS

1. Variables

Variables are one of the basic features of LESS. When using CSS, it is common to see the same value used multiple times in your stylesheets. Your code can be easier to maintain by using variables to control these values. Variables can be used to substitue for values (like px and numbers), as well as classes and elements.

To define a variable in Less, just add a ‘@’ sign before the name of your variable.

LESS ---> CSS
@default-color: #428bca; //sea blue .h1 { color: @default-color; } .h2 { color: red; background-color: @default-color; } .h1 { color: #428bca; } .h2 { color: red; background-color: #428bca; }

Properties can also be used as variables by using a ‘$’ sign before the name.

LESS ---> CSS
.h1 { color: #EFEFEF; background-color: $color; } .h1 { color: #EFEFEF; background-color: #EFEFEF; }

2. Nesting

Before LESS, every rule in CSS had to be written separately out, even if they were for parts of the same class or element. For example, if a header has multiple parts, you would have to write out its properties like so:

#header { color: red; } #header .navigation { font-size: 20px; color: blue; } #header .logo { width: 300px; color: yellow; }

However, with LESS, you can nest these separate rules inside the brackets for the parent element. This makes the code easier to read and more organized. Using the same example above, the code would now look like this:

#header { color: red; .navigation { font-size: 20px; color: blue; } .logo { width: 300px; color: yellow; } }

To reference the parent class, you can use the ‘&’ symbol inside the class. In this example, -submit and -cancel are two parent classes that each have a button class:

LESS ---> CSS
.button { &-submit { background-image: url("s.png"); } &-cancel { background-image: url("c.png"); } } .button-submit { background-image: url("s.png"); } .button-cancel { background-image: url("c.png"); }

3. Mixins

A mixin is a group of CSS properties or elements. By grouping them together, you can include a bunch of properties from one rule-set into another rule set. Ids and classes can be considered mixins. For example, we have the class .default and we want to use its properties for other classes:

.default { color: red; border-top: 10px solid black; }

To include the properties of .default in other classes, simply call it like you would for a function.* Now the #header div will have all the properties of the .default mixin.

LESS ---> CSS
.default { color: red; border-top: 10px solid black; } #header { .default(); background-color: blue; } #header { color: red; border-top: 10px solid black; background-color: blue; }

*Currently, the parentheses after the mixin call are optional.

If you don't want to use all of a mixin's properties for a class, you can call on specific properties or variables of a mixin by treating it like a dictionary key and value pair.

#default { @default-color: orange; background-color: blue; font-size: 10px; border-top: 10px solid black; } .h1 { color: #default[@default-color]; background-color: #default[background-color]; font-size: 20px; } .h1 { color: orange; background-color: blue; font-size: 20px; }

4. Parametric Mixins

Continuing with the analogy of mixins being like functions, LESS allows you to pass variables through mixins like a function. These are called parametric mixins. You can choose to have a default paramter value or not.

//parametric mixin without default value .border-radius(@radius) { -webkit-border-radius: @radius; -moz-border-radius: @radius; border-radius: @radius; } //parameter radius has default value of 5px .rounder-corners(@radius: 5px) { -webkit-border-radius: @radius; -moz-border-radius: @radius; border-radius: @radius; }

When using the parametric mixin, you put your parameter value in as you call it, just like a function.

LESS ---> CSS
//parametric mixin without default value .border-radius(@radius) { -webkit-border-radius: @radius; -moz-border-radius: @radius; border-radius: @radius; } .header { .border-radius(4px); color: red; } .header { -webkit-border-radius: 4px; -moz-border-radius: 4px; border-radius: 4px; color: red; }

In addition, you are able to define multiple mixins with the same name and number of parameters. If you use the mixin with one parameter, then properties of all mixins with that name and exactly one mandatory parameter will be used:

LESS ---> CSS
//first mixin .mixin(@color) { color-1: @color; } //second mixin .mixin(@color, @padding: 2px) { color-2: @color; padding-2: @padding; } //third mixin .mixin(@color, @padding, @margin: 2px) { color-3: @color; padding-2: @padding; margin: @margin; } .somediv { .mixin(#00800); } .somediv { color-1: #00800; color-2: #00800; padding-2: 2px; }

In the above example, using the call .mixin(#00800) in .somediv allowed it to obtain properties from the first and second mixins (color-1, color-2, and padding-2). This works because, even though the two mixins don't have the same number of parameters, in the second mixin, @padding has a default value so it is not a mandatory parameter. Since the first and second mixin both have one mandatory parameter and the function call .mixin(#00800) satisfies them both, .somediv inherits properties from both mixins. Notice that this doesn't work for the third mixin because it requires two mandatory parameters, @color and @padding, and the function call only provides one parameter value.

5. Merge

In Less, the merge feature allows for aggregating values from multiple properties into a comma or space separated list under a single property. In the example below, the .myclass wants to merge the transform properties called in the mixin with its own. By using the "+_" sign after both properties (inside the mixin and the separate class), the transform properties are merged together with spaces.

LESS ---> CSS
.mixin() { transform+_: scale(2); } .myclass { .mixin(); transform+_: rotate(15deg); } .myclass { transform+_: scale(2) rotate(15deg); }

If you want to merge using commas, use the "+" sign instead of the "+_".

6. Logical Functions

LESS supports a range of functions: logical, mathematical, string, etc. The first that we will introduce is the logical functions.

The first function is the if() function: if(condition, value1, value2): returns value1 if condition is true, else returns value2.

EXAMPLE ---> RESULT
@some: foo; div { margin: if((2 > 1), 0, 3px); color: if((iscolor(@some)), darken(@some, 10%), black); } div { margin: 0; color: black; }

The second logical function is the boolean(condition) fucntion, which takes a condition and evluates it to true and false.

EXAMPLE ---> RESULT
@bg: black; @bg-light: boolean(3 > 2); div { background: @bg; color: if(@bg-light, black, white); } div { background: black; color: white; }

7. Math Function

Next, we have the math functions. Less supports a multitude of mathematical procedures ranging from ceiling, floor, percentage, etc. Below we will list the most useful ones, along with their functions if you are not familiar with them.

  • Ceiling - rounds up to nearest integer
  • Floor - rounds down to nearest integer
  • Percentage - converts point to percentage string
  • Rounding numbers - rounds numbers to specified decimal point
  • Absolute value - gets the absolute value of a number
  • Square root - gets sqrt of a number
  • Trigonometric Functions - sin, cos, tan…
  • Pi

Since this is only an introduction to Less, we will just show you two types of functions. You can always go to the Less website to learn more about the functions that Less supports.

Advantages of LESS

There are many advantages of LESS:

  • LESS supports creating cleaner, cross-browser friendly CSS faster and easier.
  • LESS is designed in JavaScript and also created to be used in live, which compiles faster than other CSS preprocessors.
  • LESS keeps your code in modular way which is really important by making it readable and easily changeable.
  • Faster maintenance can be achieved by the use of LESS variables.
  • LESS enables you to write better and well-organized code by using nesting.
  • LESS enables you to reuse the whole classes easily by referencing them in your rule sets.
  • LESS provides the use of operations that makes coding faster and saves time.
